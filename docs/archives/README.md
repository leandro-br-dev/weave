# Archives Directory

This directory contains historical, completed, or superseded documentation that is no longer needed as active reference.

## Archive Structure

```
archives/
├── README.md                    # This file
├── dashboard/                   # Completed dashboard work (i18n fix reports)
│   └── i18n-fix-report.md
├── dark-mode/                   # Completed dark mode implementation
│   └── README.md
├── features/                    # Completed feature verification
│   └── README.md
├── i18n/                        # Completed i18n integration work
│   └── README.md
├── implementation/              # Completed implementation documentation
│   └── README.md
├── organization/                # Historical file organization/migration docs
│   └── README.md
├── testing-reports/             # Historical test reports and coverage data
│   └── README.md
└── workflow-limits/             # Completed workflow limits API changes
    └── README.md
```

## What Gets Archived

Documentation is moved here when:
- Implementation is complete and the doc served as a work log
- Features are fully integrated and verification docs are historical
- File organization/migration planning is no longer needed
- Test reports are from past sprints and superseded by current testing

## Retrieval

Archived documents can be retrieved for:
- Historical research on past implementations
- Understanding design decisions
- Bug investigation in legacy areas
- Audit requirements
