# Internationalization (i18n)

## Overview

The dashboard supports internationalization using **i18next** with **react-i18next**. The system provides full translation support for all UI components with automatic language detection and persistence.

### Supported Languages

- **en-US** (English - United States) — Default
- **pt-BR** (Portuguese - Brazil)

### How It Works

The i18n system detects language in this order:
1. **localStorage** (key: `i18nextLng`)
2. **HTML tag** (`<html lang="en-US">`)
3. **Browser** language settings

### Translation Key Structure

All translations use a single namespace with nested keys:

| Prefix | Purpose |
|--------|---------|
| `common.*` | Shared UI elements (navigation, buttons, status, messages) |
| `pages.*` | Page-specific translations (dashboard, workflows, agents, settings, etc.) |
| `components.*` | Reusable component translations (pagination, dialogs, toasts, etc.) |

### Basic Usage

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation(); // No namespace parameter
  return <h1>{t('pages.dashboard.title')}</h1>;
}
```

### Key Takeaways

1. **Single Namespace Architecture**: Uses a single namespace (`translation`) with nested keys
2. **Consistent Key Prefixes**: All keys use descriptive prefixes (`common.`, `pages.`, `components.`)
3. **Full Path Access**: Components access translations using full paths from the root
