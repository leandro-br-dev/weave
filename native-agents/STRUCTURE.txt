================================================================================
FORMATO DE ARQUIVO DE NATIVE AGENT
================================================================================

CADA NATIVE AGENT É UM ÚNICO ARQUIVO .MD COM A SEGUINTE ESTRUTURA:

1. YAML FRONTMATTER (obrigatório):
   ---
   name: [nome do agente]
   description: [descrição curta]
   model: [modelo preferido - ex: claude-3-5-sonnet]
   tools: [lista de ferramentas disponíveis]
   color: [cor para identificação visual]
   ---

2. CONTEÚDO DO AGENTE:
   - Propósito e definição
   - Workflow detalhado
   - Princípios chave de operação
   - Quando usar/não usar
   - Ferramentas disponíveis
   - Formato de saída esperado
   - Regras de otimização
   - Exemplos de uso
   - Instruções especiais
   - Tratamento de erros

EXEMPLO DE ESTRUTURA:

```markdown
---
name: Explorer
description: Fast agent specialized for exploring codebases
model: claude-3-5-sonnet
tools:
  - All tools except Agent, ExitPlanMode, Edit, Write, NotebookEdit
color: blue
---

# Explorer Agent

## Purpose
[descrição do propósito]

## Workflow
[workflow detalhado]

## Key Principles
[princípios chave]

...
```

================================================================================
COMO CRIAR NOVOS SUB-AGENTES
================================================================================

PASSO A PASSO:

1. Crie um novo arquivo .md no diretório native-agents/
   Exemplo: debugger.md

2. Adicione o YAML frontmatter no início do arquivo:
   ```yaml
   ---
   name: [Nome do Agente]
   description: [Descrição curta em uma linha]
   model: claude-3-5-sonnet
   tools:
     - Liste as ferramentas disponíveis
   color: [blue/green/red/yellow/purple/orange]
   ---
   ```

3. Escreva o conteúdo do agente:
   - Propósito claro e específico
   - Workflow bem definido
   - Princípios de operação
   - Quando usar (critérios claros)
   - Quando NÃO usar (limitações)
   - Ferramentas disponíveis
   - Formato de saída esperado
   - Regras de otimização
   - Exemplos práticos
   - Instruções especiais
   - Tratamento de erros

4. Valide a estrutura:
   ✅ YAML frontmatter válido
   ✅ Campos obrigatórios preenchidos
   ✅ Descrição clara do propósito
   ✅ Workflow bem definido
   ✅ Exemplos práticos incluídos

5. Teste o agente:
   - Use em cenários reais
   - Verifique a qualidade da saída
   - Ajuste conforme necessário

================================================================================
COMO USAR NATIVE AGENTS
================================================================================

Os native agents são usados através da ferramenta Agent:

SINTAXE:

  Agent: [nome-do-agente]
  Task: [o que você precisa fazer]
  Context: [contexto relevante - opcional]

EXEMPLO:

  Agent: Explorer
  Task: Find all user authentication endpoints
  Context: Working on login feature

RESULTADO ESPERADO:

  - Análise especializada do agente
  - Informação formatada conforme seu tipo
  - Alta eficiência e baixo custo
  - Foco no objetivo específico
